## 背景故事：太空荒原

- Frontier 是三座遥远星系相互碰撞交汇的产物。这场碰撞撕裂了空间，形成了被称为“裂隙（Rifts）”的空间裂口。
- 这些裂隙使人们能够采集一种名为“粗质物（Crude Matter）”的异质材料——它可用于先进的能源与计算技术，因此对地球人（Terrans）而言极其抢手。于是他们通过微小的空间通道投送了无数“种子”，这些种子在漫长岁月里成长为巨型工厂殖民地。历经数千年，它们最终变成了一张庞大的“幽灵基础设施”网络。
- 人类曾多次试图在 Frontier 生存与定居，但都以失败告终。不同的群体先后到来——强化殖民者、宗教邪教、军事力量……然而迟早，他们都死了：要么败于 Frontier 本身严酷的环境，要么被“野化 AI（Feral AIs）”的压力所摧毁。所谓野化 AI，是一种野生且危险的本地人工生态系统，源自最初地球人自动化体系的后裔。
- 如今，又过了数千年。当“守护之境”（Keep）遥遥瞥见了远在天边的新伊甸（New Eden）的存在——那是一个试图拯救文明的神秘实体——它开始唤醒 Frontier 的玩家角色。于是，人类文明在此重建的又一次尝试，就此开启……

## 世界轮廓

- Frontier 的底层逻辑是科学。这里的科学可以很前沿、很离奇，甚至带点推测性质（比如超光速技术），但它始终讲得通、前后自洽，不会凭空胡来。
- Frontier 是一种“远古的未来”。它明明建立在硬核科学的基础上，但生活在这里的人看它时却像隔着一层滤镜——带着神话感、原始感、神秘感，甚至有点邪门和瘆人。
- Frontier 是一座死去的世界，也是考古现场。除了玩家之外，所有人都已经死了，而且死了几千年。你面对的是遗迹、残骸、旧文明的尸体。
- Frontier 是彻底的“地狱式死亡”。它壮丽、冷漠、致命：一个逼人挣扎、也可能逼人开悟的地方。
- Frontier 是 grimpunk（阴郁朋克/灰暗朋克）。是的，你可以把尸体加工成制造材料——但那不是为了酷，也不是为了享乐，而是为了在几乎不可能的环境里活下去。

## 美术方向

- 远古的未来（Ancient future）：不是《苍穹浩瀚》（The Expanse）那种冷峻的蓝色科幻，而是更接近《沙丘》或塔图因（Tatooine）那种被烈日烘烤过的、温暖而苍茫的遥远未来。
- 整体配色偏单色系（但常常依然很鲜艳）：尽可能采用单色（或接近单色）的整体色调，这样“必要的”颜色强调元素（可自定义的玩家资产、UI、特效 VFX）才能更醒目地跳出来。
- 粗野主义 + 极简主义（Brutalism, minimalism）：为了可读性与可达性，尽量牺牲装饰性细节，但整体仍要保持克制而优雅。

## Frontier的核心循环

- 在 Frontier 里，就像在《EVE Online》里一样，核心循环围绕着一个真正由玩家驱动的经济展开：每一艘飞船、每一架无人机、甚至每一发弹药，都是由玩家使用玩家采集到的材料生产出来的。
- 此外，我们引入了“能量”和“燃料”的概念，进一步加深经济循环，并为我们在体验中更充分地拥抱自动化与 AI 打开空间——在这个体系里，每一个行动者（agent）都能成为经济中有价值的贡献者。
- Frontier 的起点是先搭建支撑核心循环所需的关键组件，并在此基础上持续以我们的聚焦愿景去改进和完善它们。
- 随着时间推移，我们会继续与社区紧密协作，不断加入新的系统与内容。
- 其中一个关键的差异点在于“破坏”的重要性——大多数游戏最终都会遭遇通货膨胀和战力膨胀（power creep）的问题，结果往往变成只服务于一个规模不断缩小的老玩家群体。
- 死亡是严肃的机制：它是“永续游戏”的一项核心设计原则。
- Each step in the core loop requires energy, a key TO FRONTIER‘S ECONOMY

## 粗质物与燃料

- Frontier里的每个行为都在为$EVE赋予价值
- 固定比例
- CCP定价
- 供应控制
- Ccp pricing
- 外部市场
- 燃料消耗
- 来自 Frontier 深空的粗质物（Crude Matter）被精炼成燃料（Fuel）——这是 Frontier 上每一次行动背后的能量来源。
- 由于每一个行动都会消耗燃料，燃料就成了支配这个世界“物理规则”的关键力量。
- 通过把“成本”与“玩法”绑定在一起，Frontier 将稀缺性、平衡性与决策的重量内置进玩家的每一次选择。
- 要开采粗质物必须使用透镜（Lenses），而透镜只能通过消耗 $EVE 获取——因此所有活动都会推动对 $EVE 的需求上升。
- 通过在游戏内对 $EVE 进行主动的供给控制、对关键游戏内物品实行垄断式定价，并强制保持稀缺性，系统将形成一种自我平衡的经济，从而确保长期可持续。

## Sandbox Access: Utopia

- Builder 需要一个可控环境来测试合约、工作流以及各种集成
- Sandbox access 是从 onboarding 进入真实开发流程的重要一环
- 如何进入 Utopia
- 右键点击 Eve Launcher 的快捷方式，并打开属性
- 在 Shortcut 页签的 Target 项最后，添加 --frontier-test-servers=Utopia
- Slash commands
- /moveme - 显示一个恒星系列表，你可以立即移动到其中任意位置
- /giveitem <itemid> <quantity> - 将指定物品生成到当前舰船的货舱中
- /giveitem "<item name>" <quantity> - 与上面功能相同，但可以直接使用物品名称，而不是 ItemID
- 一套成熟的 Builder 工作流，不仅取决于工具本身，也取决于这些工具能否在安全的环境中被验证
